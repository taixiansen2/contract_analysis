## 4. 数据质量说明

### 4.1 混合抓取管线（Dune + Helius）

- **HumidiFi swap（细节）**：用 Dune `dex_solana.trades` 拿到 SOL/USDC 成交签名清单，再以 Helius `getTransaction` 逐条拉原始数据，解析出 `state_account` / `side` / `base_amount_sol` / `quote_mint` / `price_usdc_per_sol` / `cu_consumed` / `fee`。Dune 本身不含这些链上细节，但提供了完整的成交签名列表，避免了对 HumidiFi 程序 PDA 做全量翻页。
- **HumidiFi oracle_update（方案 B）**：按池子 PDA（`--pool-accounts`）调用 `getSignaturesForAddress` 翻页，对该池 1h 内所有相关 tx 一律拉细节；本 pilot 覆盖 top 1 热门池 `8sKQHfjNhvmA...`，共 59,653 条 tx，内含 28,146 次 oracle_update（7.82/s 更新节奏）。
- **Orca/Whirlpool/Raydium（对照组）**：直接用 Dune 的 `dex_solana.trades` 已解码宽表（含 `amount_usd`、`token_bought/sold`_*），不走 Helius RPC。
- **Binance 1s K 线**：HTTP `/api/v3/klines` 按日拉取，close 价当 `cex_mid`。

### 4.2 已知限制

- Solana `block_time` 仅秒级，Binance 1s 分桶 ≥ 秒级对齐为理论上限；`delta_t_sec` 几乎都落在 0~1s，说明 HumidiFi swap 与 oracle_update 在同一秒或相邻秒发生，已达数据精度极限。
- **多跳 aggregator 的 dust 腿**：HumidiFi 在 Jupiter 等聚合器中做一个小池子（通常只是多跳路径的一腿），解析出的 `base_amount_sol` 可能 < 1e-3 SOL，这类 `price` 不具参考价值；统计时按 `is_dust=TRUE` 标记并从偏离率均值/分位计算中剔除。
- **方案 B 只覆盖 top 1 池**：其它 ~50 个活跃池的 oracle_update 只有 77s 窗口样本（约 315 条），所以 `delta_t_sec` 主要来自 `8sKQHf...` 池的 swap。继续扩大可通过 `--pool-accounts` 逗号分隔多个池。
- **Dune 摄取延迟**：`dex_solana.trades` 最新一天通常延迟 6–24h 后齐，pilot 选 2026-04-15 即为此；最新日用 Dune 跑会 0 行。
- **revert 的 swap 目前丢弃**：`tx.err ≠ None` 的 swap 未解析；如果需要统计 revert 分布，可扩展 parser。
