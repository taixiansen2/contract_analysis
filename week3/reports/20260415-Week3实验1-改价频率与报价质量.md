# 2026-04-15 · Week 3 · 实验 1：改价频率与报价质量

## 1. 研究问题 & 假设

- **研究问题**：HumidiFi 的"改价 (oracle update) 频率"和报价偏差之间有无量化关系？Solana 的链级特性是否让 HumidiFi 在高频改价中维持更好的报价质量？
- **假设（实验卡）**：HumidiFi 的 Δt（距上次成功改价的时间）越大，报价偏差越大；对照组（传统 AMM 无主动改价，用"距上次被交易的时间"作 proxy）的斜率应表现出不同规律。

## 4. OLS 回归

模型：`quote_deviation ~ delta_t_ms + const`；敏感性：`log10(quote_deviation) ~ log10(delta_t_ms + 1)`。


| amm_type  | n     | 线性 coef (bps / ms) | std_err | t     | p         | intercept (bps) | R²     | log-log slope | log-log p |
| --------- | ----- | ------------------ | ------- | ----- | --------- | --------------- | ------ | ------------- | --------- |
| humidifi  | 1,962 | +0.0039            | 0.0053  | 0.73  | **0.466** | 3.200           | 0.0003 | −0.003        | 0.894     |
| whirlpool | 3,605 | +0.00002           | 0.00009 | 0.24  | **0.808** | 3.981           | 1.6e-5 | −0.011        | 0.110     |
| raydium   | 8,338 | −0.00106           | 0.00110 | −0.96 | **0.337** | 8.810           | 0.0001 | −0.009        | 0.110     |


三家斜率都**不显著**（p ≫ 0.05，R² 近 0）。换言之：**在这个 1 小时高活跃度窗口里，Δt 对报价偏差的线性影响无法被数据区分出来**。

## 6. 结论

1. **HumidiFi 的 Δt 极度偏左**：1,962 个有效样本中，**96%（1,875 条）落在 B0 (< 400 ms)**，即 swap 发生时 oracle 在同一 slot 内刚刚更新过，B2（800-1200 ms）只有 9 条，B3 仅 1 条。这本身就是 HumidiFi 实验核心结论的一个侧面——**改价频率高到让 Δt 几乎没有横向离散度**。
2. **Δt → 偏差的线性关系在 1 小时窗口内无法被统计显著验证**：HumidiFi 的斜率是 +0.0039 bps/ms（方向与假设一致），但 p=0.47、R²=0.0003；对照组斜率也不显著。这**不等价于假设被证伪**——原因更可能是：
  - HumidiFi：样本集中在 [0, 2] slot 的极短区间（~96% 在 dt=0 slot），杠杆不足以驱动线性回归。
  - Whirlpool/Raydium：价格由 AMM 曲线决定，"距上次被交易时间"不是偏离 CEX 的主驱动（LP 再平衡、oracle 没有等等），这个 proxy 的信号很弱。
3. **跨 AMM 横向对比（不按 Δt 分桶、全体非 dust 的水平）**：HumidiFi median 0.954 bps < Whirlpool 1.217 bps < Raydium 2.528 bps；B0 桶内同样 HumidiFi 3.306 bps < Whirlpool 5.017 bps < Raydium 8.236 bps。**这个"更贴 CEX"的绝对优势和实验 0 的结论一致**。
4. **一句话结论**：HumidiFi 的改价频率已经高到让绝大多数 swap（96%）发生在与前次 oracle 同一 slot 内；在这种情况下，Δt 对偏差的线性回归系数为 +0.0039 bps/ms（不显著，p=0.47），与其说是"Δt 越长偏差越大"，不如说是"Δt 长到能被观察的样本太少"——Solana 的低 CU 成本让 HumidiFi 把 Δt 压到了统计上几乎没有方差的程度。

## 7. 已知限制

- **HumidiFi 池覆盖**：仅对 top1 池 `8sKQHf…` 跑了方案 B oracle 抓取 → 有效样本 n=1,962。扩到 top 3-5 池预计能把样本拉到 ~5k，让 B2/B3 桶有足够统计。
- **对照组 Δt 是 project 级 proxy**：没有 pool 地址，无法做"同池前一笔 swap"的严格口径；Whirlpool/Raydium 的 Δt 实际上是同项目下所有 SOL/USDC 池的混合流。
- **时间窗口短**：1 小时 pilot 在这一天是平稳市场（CEX p50 波动 ~2bps），**没有捕捉到高波动事件**；实验 3 专门研究突变事件下的响应，不在本实验验证范围。
- **Δt slot 粒度**：Solana block_time 秒级粒度；slot 粒度虽精确但 Solana 实际 slot 间距有 jitter（300-500 ms），换算 `slot × 400ms` 存在 ±25% 的抖动。
- **Orca classification**：Dune `dex_solana.trades` 在此窗下把所有 Orca 交易归到 `whirlpool`，`project='orca'` 返回 0 行；本次的 Orca CLMM 对照由 `whirlpool` 承担。
